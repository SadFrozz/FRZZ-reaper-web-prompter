## v1.5.1
### Добавлено
- Настройка «Игнорировать цвета проекта», которая по умолчанию отключает цвета дорожек/элементов в пользу карты ролей и даёт возможность вернуть исходные цвета при необходимости.
- Кнопка импорта карты ролей из проекта, собирающая роли по цветам из актуального файла subtitles.json и формирующая группы в модалке раскраски. Кнопка импорта карты ролей отображается только при наличии нескольких различных цветов в загруженных субтитрах.
- Раздел «Сегментирование» в настройках: основной тогл, плитки автоопределения/отображения субтитров с управляющими селекторами, включающий шаблон по умолчанию «$N (сери*|эпизод*|част*)» и подсказку по синтаксису. Поиск дорожек/тейков для видео-сегментации теперь требует точного совпадения по имени (без учёта регистра). А также добавлен ручной режим сегментирования с возможностью генерации сегментов.
- Статистика учитывает сегментацию по видео или маркерам (тайм-коды для сегментации берутся исходя из настроек сегментации)

### Изменено
- Добавлен парсинг Актер\Роль в виде скопированной Word таблицы (Первый столбец - Актёр, Второй Роль)
- Добавлено переименование актёров в карте ролей. Переименование актёров сохраняет формат записей карты ролей, совпадающий с результатом импорта.

### Исправлено
- Парсинг ролей\актеров, теперь все имена приводятся к капсу вне зависимости от оригинального регистра


## v1.5.0
### Добавлено
- Полноэкранный режим.
- Переключатель формата таймкода (кадры/миллисекунды/авто) с автодетектом при стабильном FPS и синхронизацией настроек UI.
- Прыжок по клику: клик по строке перемещает курсор с учётом пре-ролла и защит от воспроизведения/записи.
- Настройка отображения транспорта и таймкода, включая плавающие кнопки для раскрытия панели при сворачивании.
- Плиточный блок подсветок: отдельные тогглы для текущей/паузы/прошлой реплик и прогресс-бара, мгновенно обновляющие классы без перезагрузки.
- Переключатель режима прогресс-бара («под репликой» или «под таймкодом») с общей цветовой настройкой.
- Компактный режим навигационной панели с анимированным сворачиванием, сохранением доступа к настройкам и мобильными плавучими контролами.
- Подсветка пересечений: реплики, которые налазят друг на друга, выделяются в один блок.
- Фильтрация реплик по актёру и\или роли. Теперь можно полностью скрывать или затемнять реплики. Фильтрация по актёрам работаем в режимах MUTE\SOLO, по ролям только в режиме SOLO и может быть совмещена с фильтрацией по актёрам.

### Изменено
- Оптимизирована графика: текстовая метка «Дорожка:» заменена на иконку для экономии места в навпанели.
- Обновление данных проекта теперь дожидается статуса PENDING перед OK, проверяет метки времени JSON и статуса, выкидывая устаревшие ответы.
- Автоскролл переработан: построчный режим использует адаптивную плавность и инерцию для динамической скорости, постраничный — динамическую длительность.
- Блок настроек автоскролла вынесен отдельно, появились слайдеры для границ прокрутки в постраничном и позиции строки в построчном режимах.
- Модальное окно настроек переработано в плиточный макет: единые тогглы вместо чекбоксов, отдельные карточки для ролей и шахматной раскраски, выравнивание полей и групп.
- Навигационная панель получила классы `compact-mode` и `transport-timecode-hidden`, анимацию разворота и переработанные размеры кнопок.

### Исправлено
- Восстановлена сборка chunked-пакетов и хелпер задержки, из-за чего перестали падать первые обновления и пропадать данные на крупных проектах.
- Улучшено быстродействие для больших наборов реплик.
- Исправлена работа поиска, ранее он был ограничен только видимой областью.
- Исправлен алгоритм поиска настроенного веб-сервера в скрипте запуска.
- Исправлена подсветка прошлой реплики: классы снимаются и назначаются строго по индексу, режим паузы ведёт себя предсказуемо.

## v1.4.2
### Добавлено
- Единый запрос «Данные проекта»: фронтенд получает название, дорожки, FPS и роли одним пакетом и сразу кэширует снимок для повторных открытий.
- В окне раскраски актёров появилось предупреждение, если текущий текст не содержит префиксов [РОЛЬ].

### Изменено
- Передача больших JSON разбита на блоки и кодируется в base64, поэтому списки дорожек и роли больше не упираются в лимиты ExtState.
- FPS и имя проекта считываются напрямую из текущего проекта REAPER (учёт drop-frame и нормализация названий).
- Сохранение настроек больше не очищает ключевые слова автопоиска, если поле оставить пустым.

### Исправлено
- Парсинг ролей в модалке раскраски корректно обрабатывает сложные имена.
- Автоскрытие пустой колонки «РОЛЬ» снова срабатывает после сохранения настроек.
- Список дорожек и файл ролей больше не обрезаются на крупных проектах.
- Цвета актёров не теряются после повторного сохранения настроек.
- Таймкод на панели стабильно отображается при нестандартных FPS.
